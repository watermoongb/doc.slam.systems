DIRECTOR COR

Welcome to DIRECTOR COR, the ultimate media scheduling and playlist management application, designed exclusively for macOS 16 and above. This manual provides a comprehensive overview of the application's interface, breaking down each functional area to help you orchestrate your media seamlessly.

The application interface is divided into several logical zones, enabling an intuitive workflow from file organization to final delivery. This layout, optimized for macOS 16's modern aesthetics and performance, is divided into five key regions: the Top Bar, the Assets panel, the Playlist panel, the central Editor, and the Inspector.

Overview

Workflow Overview

This chapter describes a complete workflow from creating a new project to publishing to devices.
The example uses a Numbers spreadsheet template and multiple playback devices; you can adapt it to your own setup.

High-level steps:

  1. Create a new DIRECTOR COR project
  2. Import a playlist from a Numbers template
  3. Select the Numbers template file in the system file picker
  4. Open the Device Settings window
  5. Discover and add devices on the local network
  6. Review and manage the device list
  7. Publish and monitor sync progress
  8. Confirm publish completion

Step 1: Create a New Project

  1. From the top menu choose File → New, or use the keyboard shortcut to create a new document.
  2. After creation, the window title shows Untitled, and both the Assets and Playlist areas are empty.
  3. The Preview / Layout switch is already visible in the top-right corner, but there is no content to preview yet.

Step 2: Import from a Numbers Template

  1. In the Playlist area at the bottom-left, click the + button.
  2. In the pop-up menu, choose Import….
  3. The import feature can read an external template (such as a Numbers spreadsheet) and automatically generate playlist items and timeline entries.

If prompted: grant system permissions

During the first import, macOS may prompt you to grant permissions so DIRECTOR COR can automate Numbers and control the UI:

  • Click Open System Settings to grant Accessibility access.

  • Click Allow to permit DIRECTOR COR to control Numbers.app.

Then, in System Settings → Privacy & Security:

  • Ensure DIRECTOR COR is enabled under Accessibility.

  • Under Automation, expand DIRECTOR COR and enable control of Numbers (and other required apps, if listed).

Step 3: Choose the Numbers Template File

  1. In the system file picker, navigate to the folder that contains your template.
  2. Select the prepared Numbers template (for example HCKL_Overlay), and make sure its structure matches the DIRECTOR COR import format.
  3. Click Open. DIRECTOR COR parses the template and generates the corresponding playlist and Editor rows.

Step 4: Open Device Settings

  1. After the import has created your playlist, open the Device Settings window (typically via the Settings icon or menu).
  2. In the dialog that appears, switch to the Devices tab.
  3. At the top, choose the current Player type (for example SLAM DECK COR) so that only relevant devices are shown.

Step 5: Discover and Add Devices

  1. In the Devices window, click Add New Device to start scanning the local network.
  2. The Devices in Local Network sheet lists available players and control devices, including:
    • Device name (for example slam.systems_sev16-1-Tahoe)
    • Device type (Player or SLAM DECK COR)
    • IP address
  3. Select the devices you want to use for this project, then click Add to add them to the device list.

Step 6: Manage the Device List

  1. Back in the main Devices window you can now see all added devices:
    • Each row shows the name, IP address, version, device type, and more.
    • Use the checkboxes on the left to choose which devices should receive the current project.
  2. The action buttons on the right can be used to:
    • Test connectivity
    • Re-sync
    • Remove the device from the list
  3. When everything looks correct, click Done to return to the main window.

Step 7: Publish and Monitor Sync Progress

  1. Click the Publish cloud icon in the top-right corner to start the publish process.
  2. The Syncing with Devices window shows sync progress for each device:
    • “Syncing…” indicates that data is being transferred or processed
    • A progress bar shows the current percentage
  3. While publishing, keep the network connection stable and avoid closing the app or target devices.

Step 8: Publishing Complete

  1. When the status for all devices changes to Completed, the top of the window shows Sync completed for all devices.
  2. At this point, the playlist has been successfully distributed to all selected Player and SLAM DECK COR devices.
  3. Click Close to dismiss the dialog, then verify playback on site or in your test environment.

With these steps, you complete the full DIRECTOR COR workflow: create a project, import a template, connect devices, and publish your playlists.

Top Bar

The Top Bar is your command center, providing global navigation and control. It contains essential system and project-level functions such as project identification and publishing.

Components

  • Traffic Lights (Not Labeled): Standard macOS window controls for Close, Minimize, and Fullscreen.

  • Settings Icon (⚙️): Positioned next to the traffic lights, clicking this gear icon opens the application's global preference pane. Labeled as Settings, this is where you configure application-wide parameters, user accounts, and device connections.

  • Project Name: Displayed next to the Settings icon, this shows the current project's title. The label Project Name clarifies its purpose, allowing you to quickly identify your workspace.

  • Publish Button (Cloud with Arrow): Located on the far right, this icon, labeled Publish, is a crucial final step. It initiates the process of rendering, compiling, or uploading your completed playlist and associated media to a destination server or device. This button is your "live" switch.

ASSETS Panel

The green-tinted area on the left, labeled ASSETS, is your media library. This panel acts as a repository for all source files that can be used in your playlists. It displays a list of file names, often including metadata and file types. You can import new files, organize them into folders (folders are shown with disclosure triangles, e.g. "Europepark" and "26_MVM..."), and manage your file inventory from this view.

Usage

To add media to a playlist, you can simply drag a file or a folder directly from the ASSETS list into the central Editor table.

PLAYLIST Panel

The purple-tinted area below the ASSETS panel, labeled PLAYLIST, displays all the play sequences you have created within the current project. This is a master list of all playlists. One playlist is always selected, indicated by a dark blue background (e.g. "PLAY TIME" when selected). Other created lists such as "Animation Player," "Best Player," "Matrix," etc., are visible.

Usage

  • Selecting a different playlist from this list will immediately populate the central Editor with that playlist's specific timeline.

  • The list supports standard macOS conventions:

    • Single click to select
    • Double click to rename
    • Use the + icon at the bottom left to create new playlists, or delete existing ones

  • The bottom search bar lets you quickly filter through a large number of playlists.

EDITOR

The yellow-tinted central area, labeled EDITOR, is where you construct and manage the selected playlist's timeline. This is a spreadsheet-style table with the following columns:

ColumnDescription
StartThe exact time in the playlist when an item is scheduled to begin playing.
TimeShows the precise moment or time value related to playback for each item. This column may show the start time for the next item.
ContentThe name of the media file or command associated with that schedule block.
DurationThe total length of time that specific item will play.

Interactivity

The table is highly interactive:

  • Drag and drop media items from the ASSETS panel directly into the EDITOR to add them to the sequence.

  • Once added, you can click and drag rows to rearrange the playback order.

  • The interface automatically recalculates the start times and durations for all preceding and subsequent items to maintain a continuous, seamless playback stream.

INSPECTOR

The red-tinted panel on the far right, labeled INSPECTOR, provides detailed context-aware information and properties for the currently selected item or the overall selected playlist. It changes dynamically based on your interaction.

Preview Window

At the top of the INSPECTOR is a live preview window. When an item in the central Editor is selected, this window shows a frame from that media file or a small thumbnail of it. It can also display a live feed of the playlist's current time (e.g. a "gamma print" test screen).

Playlist Info and Layout Mode

When the overall playlist is selected (or nothing specific), the Playlist Properties and Playlist Info sections provide an overview. In Layout mode, this panel also works together with the main preview/layout switch to show how items are arranged and will appear when played:

  • Playlist Properties: Allows you to change the playlist's internal name (e.g. to "Animation Player").
  • Playlist Info: Displays a summary including:
    • Total number of items (e.g. 31)
    • Total duration of the entire playlist (e.g. 00:10:20)
    • Date and time it was last modified (e.g. 6/3/2026, 5:24 PM)

Item Properties

When a single item is selected in the Editor, the bottom portion of the INSPECTOR updates to show detailed info for that content:

  • Name: The file name of the selected item (e.g. "12_Gammaprint.mp4").
  • File Path: The absolute system path on your macOS 16 computer where the source media file is located (e.g. /Users/watermoon/Downloads/Test/Mar6_Media/12_Gammaprint.mp4). This is essential for verifying linked media or identifying missing files.

Explaination

Redundency of iPads

  1. Main purpose of iPad is control the PLAYER and show information from PLAYER. When the SPORT app quit or iPad shutdown, nothing influence to the PLAYER, PLAYER will keep stop or loop play current page of playlist automatically.

  2. When the SPORT quit or iPad reboot unexpectedly, reopen SPORT and press SYNC then select Main or Main/Backup player SPORT will show the latest information from Main PLYAER.

  3. Prepare second iPad( SPORT installed) for some emergency case (eg: iPad can't boot, overheat shutdown) is necessary.
